🔴 Breaks at runtime
Will crash or silently fail when used
CRASH
ElectionalPage — imports generateReading from gemini.js, not oracle.js
import { generateReading } from '../lib/gemini.js' — gemini.js calls the Gemini API directly but in proxy mode the key is server-side only. This call will return API_KEY is not defined or silently fail, just like the ReadingPage bug that was already fixed. Every other page uses oracle.js. Fix: change import to ../lib/oracle.
CRASH
Dashboard & TransitCalendarPage — lazy-import generateAspectReading from gemini.js
await import('../lib/gemini') then calls generateAspectReading. Same problem — goes direct to Gemini without the proxy, will throw API_KEY is not defined on click. Both pages should lazy-import from ../lib/oracle instead (which already exports generateAspectReading).
CRASH
VedicPage — imports isGeminiConfigured from gemini.js directly
import { generateVedicReading, isGeminiConfigured } from '../lib/gemini.js' — isGeminiConfigured in gemini.js was the same API_KEY ReferenceError (now fixed), but generateVedicReading is also called directly from gemini.js bypassing the proxy. Fix: import both from ../lib/oracle, which already exports both.
